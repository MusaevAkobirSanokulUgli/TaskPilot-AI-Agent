TaskPilot — Developer Use Case Guide

Everything you need to run, test, and explore the TaskPilot AI agent — from one-command Docker startup to live WebSocket inspection.

Quick Start (Docker — Recommended)

1
Navigate to project directory
cd project-3-ai-agent-taskpilot
2
Set API key (optional — rule-based mode works without it)
export OPENAI_API_KEY=sk-your-key-here
3
Build and start all 4 services
docker compose up --build
4
Open in browser and explore
See URLs below — everything is live within 60–90 seconds
Web App
http://localhost:3002

http://localhost:5002/swagger
Agent Core / Docs
http://localhost:8001/docs

http://localhost:19006

Local Development Setup (4 Layers)

Run each layer in a separate terminal tab. Start Agent Core first, then Backend, then Frontend.

Layer 1
Agent Core (Python FastAPI)
cd agent-core
python3 -m venv venv
source venv/bin/activate          # Windows: venv\Scripts\activate
pip install -r requirements.txt
uvicorn main:app --port 8001 --reload
Layer 2
Backend (.NET 8 API)
cd backend
dotnet run                        # Starts on port 5002, auto-seeds SQLite DB
Layer 3
Frontend (Next.js 14)
cd frontend
npm install
npm run dev                       # Starts on port 3002
Layer 4 (Optional)
Mobile (React Native / Expo)
cd mobile
npm install
npx expo start                    # Scan QR with Expo Go app on your phone

Testing the Agent via REST API

# Start a task (async — use WebSocket to stream events)
curl -X POST http://localhost:8001/agent/run \
  -H "Content-Type: application/json" \
  -d '{"instruction":"Triage all open bugs","project_id":"proj-001"}'

# Start a task (sync — blocks until complete, returns full result)
curl -X POST http://localhost:8001/agent/run-sync \
  -H "Content-Type: application/json" \
  -d '{"instruction":"Generate a weekly report","project_id":"proj-001"}'

# Check agent status
curl http://localhost:8001/agent/status

# Stop the running agent
curl -X POST http://localhost:8001/agent/stop

# List all available tools
curl http://localhost:8001/agent/tools

# Inspect all memory tiers
curl http://localhost:8001/agent/memory

Memory Viewer (Memory Page)

Short-Term (recent 20)
Working (active task)
Long-Term (persistent patterns)
  • Short-Term tab: last 20 interactions, sorted by recency. FIFO — oldest drops when capacity is reached.
  • Working tab: active task context in "Action: X → Result: Y" format. Cleared when agent completes a task.
  • Long-Term tab: persistent patterns with relevance scores from cosine similarity. Up to 500 entries with LRU eviction.
  • Add, edit, or delete memory entries manually from any tab.
  • Relevance scores shown next to each entry indicate retrieval priority.

  • Reports include: task completion metrics, open bugs count, team velocity, blockers, and action items.

Veb Chat Sahifasi — Rang Kodlangan Bosqich Vizualizatsiyasi

Sariq = Fikrlash (Mulohaza) Binafsha = Rejalashtirish Ko'k = Harakat (Vosita chaqiruvlari) Yashil = Kuzatish

Agent ReAct tsikli bo'ylab o'tishi bilan har bir bosqich pufagi real vaqtda paydo bo'ladi. To'liq vosita argumentlari va natijalarini ko'rish uchun har qanday qadamni bosing.

Xotira Ko'ruvchi (Xotira Sahifasi)

Qisqa Muddatli (so'nggi 20)
Ish Xotirasi (faol vazifa)
Uzoq Muddatli (doimiy naqshlar)
  • Qisqa Muddatli yorlig'i: so'nggi 20 ta o'zaro ta'sir, yangilik bo'yicha tartiblangan. FIFO — hajmga yetganda eng eski tushib qoladi.
  • Ish yorlig'i: "Harakat: X → Natija: Y" formatida faol vazifa konteksti. Agent vazifani bajarganida tozalanadi.
  • Uzoq Muddatli yorlig'i: cosinus o'xshashligi bo'yicha aloqadorlik ballari bilan doimiy naqshlar. LRU ko'chirish bilan 500 tagacha yozuv.
  • Istalgan yorliqdan xotira yozuvlarini qo'lda qo'shing, tahrirlang yoki o'chiring.
  • Har bir yozuv yonida ko'rsatilgan aloqadorlik ballari qidiruv ustuvorligini ko'rsatadi.

8가지 빠른 액션 사용 사례

1
버그 분류
"Triage all open bugs"
  • 백엔드에서 모든 열린 작업을 나열합니다
  • 심각도에 따라 우선순위를 할당합니다
  • 팀에 알림을 보냅니다
2
주간 보고서 생성
"Generate a weekly report"
  • 작업 메트릭을 집계합니다
  • Markdown 보고서를 생성합니다
  • 데이터베이스에 저장합니다
3
일일 스탠드업
"Prepare daily standup"
  • 어제의 진행 상황을 요약합니다
  • 현재 차단 요소를 식별합니다
  • 오늘의 계획을 작성합니다
4
작업 할당
"Assign unassigned tasks"
  • 모든 미할당 작업을 찾습니다
  • 팀 구성원에게 배분합니다
  • 백엔드 레코드를 업데이트합니다
5
코드 분석
"Analyze the codebase"
  • 4개 서비스 범위를 모두 스캔합니다
  • 심각도별 이슈를 반환합니다
  • 수정 제안을 제공합니다
6
상태 보고서
"Generate status report"
  • 현재 작업 상태를 읽습니다
  • 완료 메트릭을 계산합니다
  • 상태 Markdown을 생성합니다
7
차단된 작업 확인
"Check blocked tasks"
  • BLOCKED 상태의 작업을 필터링합니다
  • 차단 이슈를 분석합니다
  • 차단 해제 액션을 제안합니다
8
스프린트 리뷰
"Generate sprint review"
  • 완료된 작업 대 계획된 작업을 계산합니다
  • 속도 메트릭을 계산합니다
  • 스프린트 요약 보고서를 생성합니다

실시간 ReAct 루프 관찰

웹 채팅 페이지 — 색상으로 구분된 단계 시각화

황색 = 추론 중 (사고) 보라색 = 계획 중 파란색 = 행동 중 (도구 호출) 녹색 = 관찰 중

에이전트가 ReAct 루프를 통해 전환될 때 각 단계 버블이 실시간으로 나타납니다. 도구 인수 및 결과 전체를 보려면 단계를 클릭하세요.

대시보드 — 액션 타임라인

세션 중 에이전트가 수행하는 모든 작업에 대한 도구 이름, 인수, 성공/실패 상태를 표시합니다. SignalR을 통해 실시간으로 업데이트됩니다.

원시 WebSocket 스트림

원시 JSON 이벤트를 위해 ws://localhost:8001/agent/stream에 연결합니다: state_change, thought, plan, action, observation, complete, error, heartbeat.

메모리 뷰어 (메모리 페이지)

단기 (최근 20개)
작업 메모리 (활성 작업)
장기 (지속적 패턴)
  • 단기 탭: 최근 20개 상호작용, 최신순 정렬. FIFO — 용량에 도달하면 가장 오래된 것이 제거됩니다.
  • 작업 탭: "작업: X → 결과: Y" 형식의 활성 작업 컨텍스트. 에이전트가 작업을 완료하면 초기화됩니다.
  • 장기 탭: 코사인 유사도 관련성 점수가 있는 지속적 패턴. LRU 제거로 최대 500개 항목.
  • 어느 탭에서든 메모리 항목을 수동으로 추가, 편집 또는 삭제합니다.
  • 각 항목 옆에 표시된 관련성 점수는 검색 우선순위를 나타냅니다.

보고서 (보고서 페이지)

상태 일간 주간 스탠드업 스프린트
  • 드롭다운에서 보고서 유형을 선택하고 생성을 클릭합니다 — 에이전트를 통해 ReportGeneratorTool을 트리거합니다.
  • 보고서는 SQLite 데이터베이스에 Markdown으로 저장되고 사이드바에 나열됩니다.
  • 맞춤 Markdown 렌더러(헤더, 표, 코드 블록, 목록)로 렌더링된 보고서를 보려면 클릭합니다.
  • 보고서에는 작업 완료 메트릭, 열린 버그 수, 팀 속도, 차단 요소, 실행 항목이 포함됩니다.
