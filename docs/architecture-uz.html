🎯 Muammo ta'rifi

Nima uchun oddiy chatbotlar murakkab loyiha boshqaruvida muvaffaqiyatsiz

An'anaviy chatbotlar holatsiz va reaktiv — ular xotirasiz, rejalashtirmasdan va ko'p bosqichli harakatlar qila olmasdan har bir savolga alohida javob beradi. Loyiha boshqaruvi vazifalar, muddatlar, ustuvorliklar va jamoa a'zolari bo'yicha muvofiqlashtirish talab qiladi — bu doimiy holat va avtonom qaror qabul qilishni talab qiladi.

  • Chatbotlar rejalashtira olmaydi: ular alohida so'rovlarga javob beradi, lekin yuqori darajadagi maqsadni bajariladigan qadamlar ketma-ketligiga bo'la olmaydi.
  • Doimiy xotira yo'q: har bir suhbat yangilanib boshlanadi, foydalanuvchilar agent oldingi seanslardan eslab qolishi kerak bo'lgan kontekstni takrorlashga majbur bo'ladi.
  • Asboblarga kirish yo'q: chatbotlar aslida vazifalar yarata olmaydi, ma'lumotlar bazasini so'rov qila olmaydi, hisobotlar yarata olmaydi yoki bildirishnomalar yubora olmaydi — ular faqat bularni qanday qilishni tasvirlay oladi.
  • O'z-o'zini tuzatish yo'q: agar harakat muvaffaqiyatsiz bo'lsa, oddiy chatbot muvaffaqiyatsizlikni kuzatib, uni o'ylash va avtomatik ravishda boshqa yondashuv sinab ko'ra olmaydi.
  • Mobil va real vaqt bo'shlig'i: loyiha boshqaruvi qurilmalar bo'yicha sodir bo'ladi va push-bildirishnomalar hamda jonli yangilanishlarni talab qiladi — faqat API bilan ishlaydigan chatbotlarda bu imkoniyatlar yo'q.

Tekshirish
1. Fikrlash (Reason)

Maqsadni LLM (GPT-4o-mini) yoki kalit so'z evristikasi yordamida tahlil qilish va niyat hamda cheklovlarni tushunish.

2. Rejalashtirish (Plan)

Maqsadni kutilgan parametrlar va zahira qadamlar bilan asbob chaqiruvlarining tartiblangan ro'yxatiga bo'lish.

3. Harakat (Act)

Keyingi rejalashtirilgan asbob chaqiruvini bajarish — TaskManagerTool, CodeAnalyzerTool, ReportGeneratorTool va boshqalar.

4. Kuzatish (Observe)

Asbob natijasini yoki xatosini qabul qilish. Kuzatish hodisasini SignalR orqali ulangan mijozlarga uzatish.

5. Eslab qolish (Remember)

Yangi kuzatishlar va oraliq natijalar bilan qisqa muddatli, ish va uzoq muddatli xotira bosqichlarini yangilash.

6. Tekshirish (Check)

Maqsad amalga oshirilganligini baholash. Agar yo'q bo'lsa, yangilangan kontekst bilan Fikrlashga qaytish. Agar ha — natijani qaytarish.

Uzoq muddatli xotira

500 yozuv

Tugallangan vazifa natijalari, o'rganilgan naqshlar va vektorlar doimiy saqlash. Agentga semantik o'xshashlik qidirish orqali yangi vazifalar uchun o'tgan tajribani olish imkonini beradi.

🚀 Tezkor Boshlash

TaskPilot-ni demo ma'lumotlar bilan ishga tushirish — LLM ixtiyoriy, qoida asosidagi rejim darhol ishlaydi

1-qadam: Repozitoriyani klonlash

git clone https://github.com/your-org/ai-agent-taskpilot.git cd ai-agent-taskpilot

2-qadam: Muhitni sozlash (OpenAI kaliti ixtiyoriy)

cp .env.example .env # Ixtiyoriy: LLM asosidagi rejalashtirish uchun OpenAI kalitini qo'shish OPENAI_API_KEY=sk-... # Qoida asosidagi zahira uchun bo'sh qoldiring AGENT_MAX_ITERATIONS=10 MEMORY_LONG_TERM_CAPACITY=500

3-qadam: Docker Compose bilan barcha xizmatlarni ishga tushirish

docker compose up --build -d # Xizmatlar ishga tushmoqda: # ✓ web-dashboard → http://localhost:3002 # ✓ dotnet-api → http://localhost:5002 # ✓ agent-core → http://localhost:8001

4-qadam: API orqali agent vazifasini bajarish

curl -X POST http://localhost:5002/api/agent/run \ -H "Content-Type: application/json" \ -d '{"goal": "Foydalanuvchi autentifikatsiyasini joriy etish uchun vazifa yaratish va yuqori ustuvorlik belgilash"}' # Qaytaradi: {"sessionId": "sess_abc", "status": "running"} # ws://localhost:5002/hubs/agent da hodisalar oqimini kuzating

5-qadam: http://localhost:3002 da Kanban taxtasini oching

🔒 Xavfsizlik Masalalari

Sozlanadigan himoya choralari bilan xavfsiz avtonom agent ishlashi

🔑 API Kalit Xavfsizligi

OpenAI API kalitlari faqat muhit o'zgaruvchilarida saqlanadi. Python agent yadrosi hech qachon kalitlarni jurnallamaydi. OpenAI-ga so'rovlar faqat server tomonida amalga oshiriladi — kalitlar hech qachon brauzer mijozlariga ko'rsatilmaydi.

🔁 Iteratsiya Chegaralari

ReAct tsikli sozlanadigan maksimal iteratsiya soniga ega (standart 10). Bu LLM asosidagi rejalashtirish ishlatilganda cheksiz tsikllar va ortiqcha token xarajatlarining oldini oladi.

🛡 Asbob Kirish Nazorati

Har bir agent asbob kirishlarni bajarishdan oldin tekshiradi. TaskManagerTool maydon darajasida validatsiyani qo'llaydi. WebSearcherTool domenga ruxsat berish/bloklash ro'yxatlariga ega.

📡 WebSocket Autentifikatsiyasi

SignalR hub ulanishlari to'g'ri seans tokenini talab qiladi. Autentifikatsiya qilinmagan mijozlar faqat ommaviy hodisalarni oladi; vazifa darajasidagi hodisalar avtorizatsiyani talab qiladi.

💾 Xotira Maxfiyligi

Uzoq muddatli xotira yozuvlari foydalanuvchi seansi bo'yicha cheklangan. Maxfiy vazifalar uchun vektorlar tashqi vektorlash xizmatlariga yuborilmasdan mahalliy saqlanadi.

📱 Mobil Push-Bildirishnomalar

Expo push tokenlari shifrlangan holda saqlanadi. Bildirishnoma yuklamalari faqat vazifa ID-larini o'z ichiga oladi — to'liq vazifa ma'lumotlari bildirishnomani olgandan so'ng mijoz tomonidan olinadi.
